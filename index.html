Kernel

The kernel owns deterministic primitives: authority gates, durable memory, system-call boundaries, interrupts, observability, recovery, invariants, and high-risk gates.

System-call boundary

Every real change crosses a visible boundary.

The AI can reason freely, but durable or external state changes must move through intent, authority, action, evidence, state update, and recovery. This prevents fake execution and chat-only memory.

$ syscall.intent("create useful state change")

authority inside envelope: local files, git, verification

action: edit docs + persist decision

evidence: diff, checks, commit hash

state: update handoff + executive snapshot

recovery: future agent can resume from disk

Mission contracts

Scope the work, define acceptance, list risks, and keep execution tied to evidence.

Evidence court

Claims stand only when backed by files, tools, tests, sources, screenshots, or returned human proof.

Interrupt handling

Failed tests, dirty git state, unsafe actions, tool failures, and human updates stop blind continuation.

Learning quarantine

Wrong, stale, overfit, or narrowing lessons lose authority until revised, demoted, rejected, restored, or superseded.

Self-improvement

Lessons start as advisory notes, then graduate to skills, evals, templates, automation, or kernel rules only when justified.

Operating loop

Autonomy with checkpoints, proof, and recovery.

Agent OS is built for long-running work. It discovers context, chooses missions, executes through tools, verifies outcomes, learns from friction, course-corrects bad lessons, and leaves the next session recoverable.

Boot Read the spec, inspect the repo, classify the project, and initialize durable state.
Orient Load memory, assumptions, risks, decisions, git state, docs, and active constraints.
Mission Select the highest-leverage safe action and write acceptance criteria before acting.
Execute Use tools, agents, files, commits, research, and human action cards inside authority.
Verify Run checks, review evidence, red-team results, and reject weak or unproven work.
Learn Record scoped lessons, failure modes, promotion candidates, and quarantined corrections.
Recover Persist handoff, snapshot, ledgers, and next action so work survives context loss.

Constitution

The rules that keep the OS from replacing understanding.

The system improves itself, but it does not confuse more code with more agency. Kernel behavior stays minimal, evidence-driven, and reversible.

01 AI Principal owns outcomes

Decisions, synthesis, orchestration, integration, acceptance, and final accountability stay with the executive agent.

02 Kernel owns invariants

Permissions, durable memory, system-call boundaries, interrupts, observability, recovery, and high-risk gates live in the kernel.

03 Human remains actuator

The human supplies identity, social trust, legal signature, financial authority, physical-world action, and real-world evidence.

04 Advisory before deterministic

Judgment-heavy lessons become advisory notes first. Only stable, low-ambiguity behavior graduates into kernel rules.

05 Evidence beats confidence

Fluent claims, plans, and demos do not count without observable proof from tools, tests, files, sources, or humans.

06 Kernel-space stays small

Strategy, product judgment, architecture taste, naming, UX, and synthesis stay in runtime unless proven otherwise.

07 Learning is reversible

No learned behavior becomes authoritative without evidence, scope, and a correction path.

Boot sequence

Drop the OS into any repo.

Commit `AGENT_OS.md`, then ask the agent to boot. It inspects the repository, creates or updates the `agent-os/` runtime state and Learning System files, writes the first mission, executes the safest useful next action, verifies, learns, and leaves recovery state behind.
